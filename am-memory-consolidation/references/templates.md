## MEMORY.md Project Section Structure

When restructuring MEMORY.md during first run, organize the Projects section as:

```markdown
## 🏗️ Projects

### <Project Name 1>
<!-- All facts, decisions, architecture, status for this project -->

### <Project Name 2>
<!-- All facts, decisions, architecture, status for this project -->
```

Each project subsection should contain the project's specific knowledge.
Cross-project information (shared infrastructure, team-wide decisions) stays in
the parent section or other top-level sections (Strategy, Key Decisions, etc.).
